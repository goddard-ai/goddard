# Proposals

This directory stores approved, initiative-level proposal documents linked to the repository `README.md` and `spec/vision.md`.

Once a proposal is fully implemented, it should be deleted from this directory.

## Getting started with proposals

1. Review the existing proposals in this directory.
2. If you have a new initiative, draft a proposal as a Pull Request.
3. Once approved, merge it into this directory.

## Naming and status conventions

- Proposal filenames must use this format: `YYYYMMDDHH_kebab-case-title.md`
  - Example: `2026030318_extension-governance-update.md`
- Proposals must only be merged here when approved. Unapproved drafts and rejected proposals exist as pull requests.

## Proposal format

Each proposal should be concise and include:

1. Problem statement
2. Proposed scope
3. Constraints and non-goals
4. Success metrics
5. Review/approval status
